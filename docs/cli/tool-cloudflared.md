# `skiller cloudflared`

Manage the Cloudflare tunnel used to expose the local webhooks server.

## Commands

```bash
skiller cloudflared login start
skiller cloudflared login status
skiller cloudflared login stop

skiller cloudflared ensure --domain <domain>

skiller cloudflared start
skiller cloudflared status
skiller cloudflared stop
```

Defaults:
- tunnel name: `skillerwh`
- public hostname: `skillerwh.<domain>`
- local origin: `http://127.0.0.1:8001`

Out of scope:
- deleting the remote tunnel
- deleting DNS records

## Effective HOME

All commands use the process `HOME`.

With isolation:

```bash
SKILLER_DEBUG_HOME=/tmp/skiller-cloudflared-home skiller cloudflared login status
```

## Local State

Skiller state lives under:

```text
~/.skiller/cloudflared/
```

Files:
- `login.json`: managed login state
- `login.log`: login output
- `managed-<port>.json`: managed connector ownership

`cloudflared` still uses its own files under:

```text
~/.cloudflared/
```

for example:
- `cert.pem`
- tunnel credentials
- `<tunnel>-config.yml`

## Typical Flow

1. Check login:

```bash
skiller cloudflared login status
```

If needed:

```bash
skiller cloudflared login start
```

2. Ensure remote resources:

```bash
skiller cloudflared ensure --domain campino.me
```

This ensures the tunnel `skillerwh` and the DNS route `skillerwh.<domain>`.
It also writes the tunnel config used by `start`:

```text
~/.cloudflared/skillerwh-config.yml
```

3. Start the local origin and connector:

```bash
skiller server start
skiller cloudflared start
```

`start` manages only the local connector. It does not create remote resources.
It uses the config generated by `ensure`.

4. Inspect local state:

```bash
skiller cloudflared status
```

Important fields:
- `running`
- `managed_by_skiller`
- `origin_url`
- `tunnel_name`
- `pid`

5. Stop the managed connector:

```bash
skiller cloudflared stop
```

`stop` only stops a connector owned by Skiller.
